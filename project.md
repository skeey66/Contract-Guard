
# 로컬 LLM 및 RAG 기술을 활용한 보안 특화형 AI 계약서 검토 및 독소 조항 방지 시스템 개발
- **개요:**
    - 최근 생성형 AI 도입으로 법무 업무 효율성이 증대되고 있으나, OpenAI 등 외부 서버 전송 방식은 기업의 핵심 자산인 계약서 데이터 유출 및 민감정보 노출 가능성과 컴플라이언스(국외이전/위탁 등) 부담을 수반함.
    - 외부 API 호출 없이 로컬 환경(On-Premise)에서 동작하며, 오픈소스 LLM과 RAG(Retrieval-Augmented Generation) 기술을 결합하여 계약서 내 리스크 조항(불공정 가능성, 협상 필요성, 표준 대비 불리함 등) 탐지 및 표준 약관 대조를 자동화하는 보안 특화형 리걸테크 솔루션을 구축하고자 함.
    - 법률 전문가의 개입 전, 반복적이고 정형화된 리스크 검토 업무를 자동화하여 효율성을 극대화하는 것을 목표로 함.
- **목표:**
    1. 외부 네트워크 차단 환경에서 구동되는 경량화된 오픈소스 LLM(Large Language Model) 환경 구축.
    2. RAG 기술을 적용하여 실제 표준 약관 및 근거 문서 인용 기반 답변 생성 기능 구현.
    3. 특정 계약 유형(임대차, 근로, 용역 등) 위험 조항 자동 하이라이트 및 수정 대안 제시.
    4. 법률 전문가 및 일반 사용자가 즉시 분석 결과를 확인할 수 있는 웹 기반 대시보드(UI/UX) 구현.

---

### **3. 문제의 구현(해결) 방안**

- **LLM 모델 선정 및 배포:** Ollama 또는 llama.cpp 등 로컬 배포 가능한 오픈소스 모델(Llama 3, Mistral, Solar 등) 중 한국어 성능이 우수한 모델 선정 및 양자화 배포.
- **데이터 파이프라인(RAG):** Vector DB(ChromaDB 또는 FAISS)를 사용하여 지식베이스 구축.
    - Embedding: Hugging Face의 오픈소스 모델(예: jhgan/ko-sroberta-multitask 등) 활용.
- **프롬프트 엔지니어링 및 워크플로우:** RAG를 통해 검색된 관련 조항을 컨텍스트로 주입. [근거 조항 인용 → 표준 대비 차이점 분석 → 리스크 사유 요약 → 권고 수정안 제시 → 신뢰도/근거 링크] 단계의 파이프라인 설계.
- **프론트엔드/백엔드:** FastAPI(API 서버), Streamlit(프로토타입) 또는 React/Next.js를 활용한 웹 인터페이스 구현.

### **4. 상세 요구사항**

1. **데이터 전처리 및 지식베이스 구축:**
    - 공정거래위원회 표준약관 및 AI-Hub 약관 데이터셋 수집 및 정제.
    - 사용자 업로드 계약서(PDF/HWP/Docx) 텍스트 추출 엔진 구현.
2. **로컬 LLM 튜닝 및 RAG 연동:**
    - 4-bit/8-bit 양자화(Quantization) 기술을 적용하여 일반 워크스테이션급 성능 유지.
    - 사용자 쿼리에 따른 정교한 RAG 워크플로우 구현.
3. **리스크 조항 탐지 알고리즘 최적화:**
    - '위약금 과다', '일방적 해지권' 등 주요 위험 키워드 및 문맥 기반 탐지 로직 고도화.
4. **보안성 및 검증:**
    - 외부 API 호출 차단 여부 확인 및 로컬 서버 환경에서의 성능 벤치마킹.
    - 법률 전문가 피드백을 통한 AI 분석 결과의 정합성 평가.

---

### **5. 구현 시 필수 요건**

- **데이터셋:** AI-Hub 법률/규정 데이터, 공정거래위원회 표준약관 아카이브.
- **기술서:** Lewis et al. (2020) RAG 논문, Upstage Solar-10.7B Tech Report 등.
- **최종 결과물(제출물):**

    2. 인덱싱 완료된 벡터 DB 파일/폴더.
    3. 프론트엔드 어플리케이션 소스 및 실행 파일.
    4. 리스크 탐지 정확도 및 답변 근거 일치율 테스트 결과 보고서.
    5. 개발 환경 구축 및 서비스 실행 매뉴얼.
